# Project Brief: Data Reliability Suite

## Mission
Build a production-style Data Quality & ETL Validation Framework for portfolio and interview demonstration.

The framework should simulate a real data platform:

Raw data → ETL transformation → warehouse load → automated validation → reports → CI/CD.

## Target Roles
This project should position the owner for:
- Data QA Engineer
- QA Automation Engineer in Data Teams
- Platform QA Engineer
- Data Reliability Engineer
- AI/Data Quality Engineer
- SDET with backend/data focus

## Core Principles
- Python-first
- Clean architecture
- Modular validations
- Minimal but realistic implementation
- Easy to explain in interviews
- CI/CD-ready
- Portfolio-grade documentation

## MVP Scope
Build V1 with:
- CSV ingestion
- Transformation layer
- PostgreSQL warehouse loader
- pytest validation suite
- Data quality checks
- HTML/Allure-style reporting
- Docker Compose
- GitHub Actions workflow
- Strong README

## Future Scope
Add later:
- Great Expectations optional integration

- Kafka/event validation
- AI-assisted anomaly detection
- LLM-generated data-quality summaries
- Grafana dashboard
